## б) Постановка задачи

Разработать консольное приложение на языке Python для автоматизации учета бронирований гостиничных номеров. Система должна обеспечивать полный цикл работы с данными: загрузку, создание, редактирование, удаление и долгосрочное хранение информации о ключевых сущностях гостиничного бизнеса.

### Функциональные требования

**Управление справочниками:**
- Гостиницы (название, адрес, рейтинг, телефон)
- Номера (номер комнаты, цена за ночь, вместимость, этаж, удобства)
- Гости (ФИО, телефон, email, бонусные баллы)

**Формирование бронирований:**
- Привязка бронирования к конкретной гостинице, номеру и гостю
- Указание даты заезда и даты выезда
- Автоматическая проверка на конфликты (занятость номера в указанный период)
- Автоматический расчет стоимости с учетом скидки (10% при бронировании от 7+ ночей)

**Хранение данных:**
- Автоматическое сохранение в файлы формата JSON
- Загрузка существующих данных при запуске приложения
- Корректная обработка отсутствия файла или повреждённых данных

**Пользовательский интерфейс:**
- Текстовое консольное меню с чёткой навигацией
- Валидация пользовательского ввода на каждом этапе
- Структурированный вывод для удобства восприятия

## в) Анализ задачи и структура классов

### 1. Предметный анализ

Домен системы описывается тремя базовыми сущностями: **Гостиница**, **Номер**, **Гость**. Каждая обладает уникальным идентификатором и набором атрибутов с жёсткими типами.

**Бронирование (Booking)** выступает составной сущностью, агрегирующей идентификаторы базовых сущностей и временные параметры (дата заезда, дата выезда).

### 2. Применение принципов ООП

| Принцип | Реализация в проекте |
|---------|----------------------|
| **Инкапсуляция** | Все атрибуты моделей и менеджеров объявлены приватными (префикс `_`). Доступ осуществляется только через геттеры и сеттеры |
| **Наследование** | Базовый класс `BaseEntity` выносит общую логику. Базовый класс `Person` определяет абстрактный метод `get_info()`, который реализуется в `Guest` |
| **Полиморфизм** | Все сущности реализуют одинаковые методы `to_dict()`, `from_dict()`, `__str__()`. Репозиторий работает с любым типом через Generic |

### 3. Иерархия классов

### 4. Краткое описание классов

| Класс | Назначение | Ключевые методы |

| `ConsoleApp` | Консольный интерфейс | Цикл меню, маршрутизация |

1. Скопируйте файлы проекта в рабочую директорию

Пример рабочего процесса

### Хранение данных


Все данные сохраняются в папке data/ в отдельных JSON файлах:

## hotels.json — гостиницы и номера:




```json
  {
    "hotel_id": "H001",
    "name": "Гранд Отель",
    "address": "Москва, ул. Тверская, 1",
    "rating": 4.5,
    "rooms": [
      {
        "room_number": "101",
        "price_per_night": 5000,
        "capacity": 2,
        "floor": 1
      }
    ]
  }
```





## guests.json — гости:




```json
  {
    "guest_id": "G001",
    "name": "Иванов Петр Сергеевич",
    "contact": "+7-999-123-45-67",
    "email": "ivanov@example.com",
    "loyalty_points": 150
  }
```



## bookings.json — бронирования:




```json
  {
    "booking_id": "H001_101_G001_20260101120000",
    "hotel_id": "H001",
    "room_number": "101",
    "guest_id": "G001",
    "check_in": "2026-06-01",
    "check_out": "2026-06-05",
    "total_price": 20000,
    "status": "active"
  }
```





Файлы создаются автоматически при первом запуске. При повреждении JSON или отсутствии файлов система инициализируется с пустым состоянием, не прерывая работу.

### Расчет стоимости бронирования



Стоимость = Цена_за_ночь × Количество_ночей



Количество_ночей = Дата_выезда - Дата_заезда (в днях)



Если Количество_ночей >= 7:
    Стоимость = Стоимость × 0.9  (скидка 10%)
